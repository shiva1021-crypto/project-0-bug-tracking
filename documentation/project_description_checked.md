# Checked Project Description

## Corrected Project Title

**Software Bug Tracking and Reporting Tool**

## Corrected Project Description

This project focuses on developing a web-based application named **Software Bug Tracking and Reporting Tool**. The application provides a clear and structured way for users to report, manage, assign, monitor, and resolve software defects.

The system includes role-based access for administrators, project managers, developers, and testers. Testers can submit detailed bug reports with information such as title, description, severity, priority, reproduction steps, and screenshots. Project managers and administrators can assign reported bugs to developers. Only the assigned developer can update the progress of a bug through different stages such as Open, In Progress, Resolved, and Closed.

The application is developed using HTML, CSS, JavaScript, Python Flask, and MySQL. It also includes search, filtering, dashboard statistics, comments, screenshot uploads, report generation, CSV export, PDF print export, charts, category analytics, audit history, tenant isolation, optional email notifications, external issue links, dark mode, CSRF protection, secure headers, pooled database connections, production-style WSGI support, and admin user management. These features support effective defect management and improve the software maintenance process.

## Corrected Objectives

### Essential Objectives

- To create user registration and login features so users can securely access the application.
- To implement role-based access for administrators, project managers, developers, and testers.
- To create a bug reporting feature where users can submit detailed bug reports.
- To store bug details such as title, description, priority, severity, status, reporter, assigned developer, and creation date.
- To allow authorized users to view reported bugs.
- To allow administrators or project managers to assign bugs to developers.
- To allow only the assigned developer to update the status of a bug.
- To add search and filter options for status, priority, severity, date, and assigned developer.
- To use a MySQL database to store user, role, bug, assignment, comment, and history information.
- To create a clear and usable web interface using HTML, CSS, JavaScript, and Python Flask.
- To use Python Flask for routing, authentication, backend logic, and database communication.
- To validate user input before storing data in the database.

### Desirable Objectives

- To create a dashboard showing key bug statistics such as total bugs, open bugs, resolved bugs, closed bugs, and critical bugs.
- To add a comments section so users can write progress notes on individual bug reports.
- To add screenshot upload functionality for bug reports.
- To create reporting features based on bug status, priority, severity, assigned developer, and date range.
- To use client-side and server-side validation to improve data quality and security.
- To make the application responsive for different screen sizes.
- To store timestamps for bug creation and bug updates.
- To store user passwords securely using password hashing.
- To protect restricted pages so only authenticated users can access them.
- To display clear success and error messages for user actions.

### Optional Objectives

- To add email notifications when a bug is assigned or updated.
- To export reports as PDF or CSV files.
- To include graphical charts that show bug trends and project progress.
- To create an audit trail showing the history of changes made to each bug.
- To build an advanced admin panel for managing users, roles, and permissions.
- To add dark mode or other interface customization options.
- To add integration with GitHub or another external issue tracking platform.
- To include analytics for identifying common bug categories or recurring issues.
